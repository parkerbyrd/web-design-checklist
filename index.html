HTML

  • Always close tags
  • Indent correctly
  • Avoid inline css in html file
  • Use ID's and Classes correctly
  • Dont name classes things that are already tags (header, link, etc.)
  • Use H tags! (Heirarchy)
  • Code special characters
  • Validate the code
  • SEO and links to CSS and JS files go in the head tag
  • Use UL's for the nav bar
  • Give images an "alt"

CSS

  • Use a reset. Always start with that
  • Use margins and padding and keep them responsive
  • Validate the code
  • Think minimal: dont go div crazy
  • Don't confuse floats and display
  • Name classes and IDs based on their functionality
  • Use Comments to help reference things later
  • Use Shorthand whenever possible
  • Use the return key and indents to give styles their own lines

File Management

  • useCamelCase
  • No spaces to avoid "%20"
  • Letters and numbers are okay
  • Organize project folders so links work
  • Dont name files things like "as_of_tuesday". Keep it professional

Content

  • Use grids, blocks, columns to lay out content
  • Organize content
  • Keep a consistent use of colors, fonts and hover states
  • No errors (spelling, code, design)
  • Keep it functional

Grid

  • Use grids to align content
  • Is the grid responsive?
  • Use grids to create columns

Typography

  • 40-70 characters for each line
  • Colors should contrast well to improve readability
  • Use fonts that compliment each other
  • Use Whitespace and line height

Color Theory

  • Right contrast where need be
  • Colors should all align to the theme of the site
  • Create secondary palette based off other saturations and hues or primary colors
  • Be Consistent

Using style tiles

  • Color Palette
  • Icon example
  • Show all examples of typography heirarchy
  • Do not let it look like a webpage
  • No real content. only style

Saving images correctly

  • Always use Save For Web
  • Use the right file type based on need and use of photo
  • Smaller file size, Faster loading time
  • Watch out for transparency issues

Accessibility/usability

  • Does the site load in 3 seconds?
  • Text is easy to read and flows with natural eye movements
  • Navigation is obvious and easy to use
  • Design for the colorblind too!
  • Browser Support
  • Is the site responsive?
  • Mobile first: are there elements on the page that hurt the UX on a smaller screen?

Code Validation

  • Test the Html and CSS using code validators
  • Test certain functions using Can I Use?
  • Test on different browsers and screens

Git and Github

  • Use the alphabetical order rule for git
  • Be aware of what folder and git repo you are in
  • Commit randomly
  • Remember mkdir and touch for creating project folders and files

Sass

  • Always remember to have Sass watch on
  • Use variables and mixins to shorten code and make future changes easy
  • Use import for resets and other scss files when using OOCSS
  • Nest the elements, but keep it consistent or certain changes will not occur
  • Never touch the actual css file, sass will change it for you

Command Line

  • Use "Git status, add, commit, push"
  • Remember to use "sass --watch ...scss" while working on a project
  • Git checkout changes repos
  • Use "python -m SimpleHTTPServer" to create a local server to view work on
  • Never use git init in the home folder

Responsive Web Design

  • Use percents for widths, and left to right adjustments
  • Use pixels or ems for heights and top to bottom adjustments
  • Use media queries to change things as the viewport size changes
  • Always include the viewport meta tag
  • Keep the layout, images, type, and videos flexible

Javascript

  • Link to the JS in the head using a script tag
  • The DOM is a programming interface for HTML and XML
  • You can write JavsScript inline within a script element on the html document
  • Always use var and remember the space after it
  • Get aquainted with falsy values (null, 0, undefined)
  • Empty quotes create an empty string
